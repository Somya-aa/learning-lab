# What is Git?

Git is a **Distributed Version Control System (DVCS)** that helps developers track changes in their source code during software development. It allows you to save different versions of your project, collaborate with other developers, and restore previous versions whenever needed.

Git was created by **Linus Torvalds** in **2005** to support the development of the Linux kernel. Today, it is one of the most widely used version control systems in the software industry.

---

## 📖 What is Version Control?

Version control is a system that records changes made to files over time. It enable you to:

- Track every modification.
- Restore previous versions of files.
- Compare different versions.
- Collaborate with multiple developers.
- Prevent accidental loss of work.

Think of version control as a **history book** for your project, where every important change is recorded.

---

## 🤔 Why Do We Need Git?

Imagine you're building a website.

Without Git:

- if you make a mistake, you may lose your previous work.
- Sharing files with teammates becomes confusing.
- Keeping multiple copies like `project_final`, `project_final_v2`, or `project_latest` quickly becomes difficult to manage.

With Git:

- Every change is recorded.
- You can restore any previous version.
- Multiple developers can work on the same project simultaneously.
- You can experiment safely using branches.

---

## 🔄 Types of Version Control Systems

There are three types of version control systems.

| Type | Description |
|------|-------------|
| Local Version Control System | Stores version history only on your local computer. |
| Centralized Version Control System (CVCS) | Uses a central server where everyone stores and retrieves project versions. |
| Distributed Version Control System (DVCS) | Every developer has a complete copy of the project's history. Git belongs to this category. |

---

## 🌍 Why Git is a Distributed Version Control System

Unlike centralized systems, Git stores the complete repository history on every developer's computer.

This means you can:

- Work offline.
- Commit changes without an internet connection.
- Recover the project even if the remote repository becomes unsvsilsble.
- Clone repositories with their complete history.

## 🌟 Real-World Example

Imagine you are writing a research paper.

Every day, you save a new copy:

```
paper.docx
paper_final.docx
paper_final_v2.docx
paper_final_latest.docx
paper_final_latest2.docx
```

After a few days, it becomes difficult to know which file is the correct one.

Git solves this problem by storing every version in a single repository while maintaining a complete history of chnages. Instead of creating multiple files, you simply create commits that different versions of your work.
